/pr-buddy @0.4.1

Understand the PR before you approve it.

Install GitHub

The problem

AI review tools make it too easy to approve code you haven't read. Click through, apply, merge. Later you can't explain it.

Solution

/pr-buddy <pr> walks you through a PR in 5 stations; you answer first, then the skill shows its reading.
Once you can explain the PR confidently, it runs pr-review-toolkit anchored to what you did discover.

The output is a combined PR comment that's yours to post.

Install

claude skill add kiss-skills/pr-buddy

Requirements: gh CLI (authenticated) · pr-review-toolkit for Phase 2

The 5 stations

You answer first. Then the skill shows its reading.

01
Goal
"What problem do you think this PR is solving?"
02
Architecture
"Which file is the entry point? Walk me through what you think is happening."
03
Decisions
"The author chose X. What would you have done differently?"
04
Risk
"What's the one thing that could go wrong here?"
05
Ownership
"Could you explain this PR to a teammate? What are you still unsure about?"
